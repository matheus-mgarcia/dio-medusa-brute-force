# dio-medusa-brute-force
Documentação do Lab Project "Simulando um Ataque de Brute Force de Senhas com Medusa e Kali Linux" da plataforma Dio.me

# Descrição do projeto:
O projeto documentado se trata de três pentests realizados em diferentes tipos de serviços com técnicas de brute force utilizando do software "Medusa" que está disponível por padrão nos sistemas do Kali Linux.
O primeiro pentest realizado foi um que utilizou o método de brute force contra o serviço FTP, o segundo utilizou o brute force em um formulário HTTP e o ultimo ultilizou técnicas de password spraying e enumeração em um serviço SMB

## Ferramentas utilizadas:
Para a execução do projeto foram utilizadas algumas ferramentas gratuitas para a simulação da maquina vunerável que foi invadida, o ambiente na qual foi efetuada o processo de invasão, os softwares que foram utiizados para realizar a invasão e uma ferramenta que simula um sistema web com falhas de segurança para simular uma invasão a um sistema web. São essas ferramentas:
- Oracle VirtualBox, que foi o software utilizado para a criação das maquinas virtuáis (https://www.virtualbox.org/);
- Kali Linux, que foi o sistema operacional utilizado como ambiente de trabalho para a efetuação das invasões (https://www.kali.org/);
- Metasploitable 2, que foi o sistema propositalmente comprometido utilizado para os testes de penetração (https://sourceforge.net/projects/metasploitable2/);
- Medusa, que foi o software utilizado para a realização dos testes de penetração via brute force (http://foofus.net/goons/jmk/medusa/medusa.html);
- Nmap, que foi o software utilizado para a realização do processo de enumeração dos testes de penetração (https://nmap.org/);
- DVWA, que foi a ferramenta utilizada para simular o sistema web desprotegido (https://www.dvwa.co.uk/).

## Preparações iniciais:
Antes da realização dos pentests, foi necessário a instalação do Oracle VirtualBox e também a criação das maquinas virtuais, uma para o Kali Linux e outa para o Metasploitable 2. Além disso foi prciso modificar as configurações de ambas as maquinas para que elas pudessem se comunicar em uma rede interna (host-only).

## Pentest de força bruta em FTP:
Esse pentest tem como objetivo entrar no sistema por meio de um perfil de usuário que possui uma senha fraca utilizando do protocolo FTP que está mal configurando no sistema invadido.
1. Primeiro, foi necessário verificar a conexão com o sistema alvo e para isso foi utiliizado o comando `ping -c 4 ip_alvo`, onde o comando `ping` enviará um sinal para o sistema alvo e retornará uma resposta caso tenha encontrado ela, o parâmetro `-c 4` indica a quantidade de pacotes que devem ser enviado para o sistema alvo e o `ip_alvo` no caso seria substituído justamente pelo ip do sistema alvo;
2. Após confirmada a conexão, foi utilizado o Nmap para a efetuação da enumeração do sistema alvo através do comando `nmap -sV -p 21,22,80,445,139 ip_alvo`, onde o comando `nmap` chama o programa, o parâmetro `-sV` indica para o programa que ele deve retornar as informações relacionadas as versões dos serviços que estão rodando e cada uma das portas verificadas e o parâmetro `-p` seguido pelos valores `21,22,80,445,139` indica quais as portas que devem ser analizadas pela enumeração. Com o resultado da enumeração feita durante esse passo, o nmap retornará as portas de rede abertas no sistema alvo e também a versão dos protocolos, como esse pentest está sendo realizado em um sistema propositalmente inseguro, todas as portas verificadas no comando devem estar abertas;
3. Com o resultado da enumeração do Nmap, foi feita a verificação de se as portas realmente estão abertas tentando se contectar com o sistema por elas. Como esse pentest tem como foco o protocolo FTP, foi utilizado o comando `ftp ip_alvo`;
4. Depois de confirmar a conexão, foram criadas duas wordlists contendo possíveis nomes de usuários e senhas comuns que serão utilizadas durante o processo de brute force no protocolo FTP. Os comandos utilizados para a criação dessas wordlists foram `echo -e "user\nmsfadmin\nadmin\nroot" > users.txt` para a criação da wordlist de usuários e o comando `echo -e "123456\npassword\nqwerty\nmsfadmin" > pass.txt` para a criação da wordlist de senhas;
5. Com as wordlists de usuários e senhas criadas, foi utilizado o programa medusa para a efetuação do brute force através do comando`medusa -h ip_alvo -U users.txt -P pass.txt -M ftp -t 6`, onde `medusa` e a parte do comando que chama o software, `-h ip_alvo` é onde informamos ao programa o ip do sistema alvo, `-U users.txt` é onde foi informado a wordlist de usuáios que será utilizada, `-P pass.txt` é onde foi informado a wordlist de senhas, `-M ftp` informa o método de conexão e o `-t 6` infrma o número de threads que serão utilizadas pelo programa durante o processo de brute force. O programa então retornou o resultado do brute force indicando o tipo de sucesso de cada combinação das wordlists de usuários e senhas;
6. Ao final do pentest, foi efetuada a conexão ao sistema alvo e vou testada a combinação de usuário e senha que obteve sucesso com o brute force do medusa.

## Pentest de força bruta em formulários de login web:
Esse pentest tem como objetivo entrar em um sistema web que utiliza formulários de login sem sistemas de proteção contra brute force utilizando a ferramenta medusa e o sistema DVWA como sistema alvo.
1. Primeiro foi realizada a análise da página com o formulário de login utilizando as ferramentas de desenvolvedor que existem dentro no próprio navegador para verificar a sessão "network do site" para observar o comportamento do site após o envio do formulário, os parâmetros que o servidor espera receber e o retorno de falha da página;
2. Depois de saber os campos que precisam ser preenchidos no formulário de login (que no caso do DVWA seriam apenas o campo "username" e "password"), foram criadas duas wordlists com possíveis termos para preencher os campos através dos comandos `echo -e "user\nmsfadmin\nadmin\nroot" > users.txt` para a wordlist de usuários e `echo -e "123456\npassword\nqwerty\nmsfadmin" > pass.txt` para a wordlist de senhas;
3. Após criadas as wordlists, foi utilizado o comando `medusa -h ip_alvo -U users.txt -P pass.txt -M http \ -m PAGE:'/dvwa/login.php' \ -m FORM:'username=^USER^&password=^PASS^&Login=Login' \ -m 'FAIL=Login failed' -t 6`. O início do comando `medusa` chama o programa que irá realizar o brute force, o `-h ip_alvo` informa ao programa qual é o endereço do ip do sistema alvo, o `U users.txt` informa ao programa a wordlist de usuários, o `-P pass.txt` informa ao programa a wordlist de senhas, o `-M http` indica ao programa qual a forma de conexão, o `-m PAGE:'/dvwa/login.php'` indica qual a página em que o formulário de login se encontra, o `FORM:'username=^USER^&password=^PASS^&Login=Login'` informa o que deve ser preenchido em cada campo, o `-m 'FAIL=Login failed'` informa o que será retornado pela página em caso de falha e o `-t 6` informa o númer de threads que podem ser utilizadas durante o processo de brute force pelo programa. O retorno do programa foram as combinações de termos nas wordlists e o tipo de sucesso que as combinações tiveram (obs: após encontrar uma combinação que funcione, o medusa informará que todas as outras combinações após o primeiro sucesso também são um sucesso);
4. Ao final do pentest, foi efetuado o login utilizando a combinação de usuário e senha que foi retornado pelo software medusa em seu processo de brute force na página com formulário de login.

## Pentest com Enumeração e Password Spraying em SMB:
Esse pentest tem como objetivo invadir um sistema pelo serviço SMB através de técnicas de enumeração e password spraying tendo como alvo o sistema do metaploitable;
1. Primeiro foi feita a enumeração dos usuários do sistema alvo através do comando `enum4linux -a ip_alvo | tee enum4output.txt`. O resultado desse comando é a criação de um documento .txt contendo todos os usuários do sistema alvo entre outras informações. É possível abrir o arquivo para ser visualizado pelo terminal através do comando `less enum4_output.txt`;
2. Apatir das informações adquiridas através da etapa anterior, foram criadas as wordlists de usuários e de senhas através dos comandos `echo -e "user\nmsfadmin\nservice" > smb_users.txt` para usuários e `echo -e "password\n123456\nWelcome123\nmsfadmin" > senhas_spray.txt` para senhas;
3. Com as wordlists criadas, foi utilizado o comando `medusa -h ip_alvo -U smb_users.txt -P senhas_spray.txt -M smbnt -t 2 -T 50`, onde `medusa` chama o software que irá realizar o password spraying, `-h ip_alvo` informa o ip do sistema alvo, `-U smb_users.txt` informa a wordlist de usuários, `-P senhas_spray.txt` informa a wordlist de senhas, `-M smbnt` informa o serviço que será utilizado no ataque, `-t 2` informa o número de threads que serão utilizadas no processo e `-T 50` informa o número de hosts em paralelo. O retorno desse comando é o tipo de sucesso que cada combinação de usuário e senha teve ao realizar o password spraying;
4. Ao final do pentest, foi verifcado se a combinação de usuário e senha que obteve sucesso na etapa anterior realmente funciona utilizando o comando `smbclient -L //ip_alvo -U usuario_sucesso` onde `smbclient` é o comando que tenta conexão com o serviço SMB, `-L //ip_alvo` informa o ip do sistema alvo e `-U usuario_sucesso` informa o usuário nome da conta de usuário no qual está tentando ser efetuada a invasão. Depois, o comando exigirá a senha do usuário então foi colocada a senha que obteve sucesso na etapa anterior, finalizando assim o pentest.

## Conclusão:
O que podemos concluír com esse projeto é como a má configuração das portas de um sistema, a negligencia em relação a senhas fortes por parte dos usuários e a falta de um sistema de segurança mais robusto podem resultar em brechas na segurança que qualquer idivíduo sem muita esperiência mas com as ferramentas certas pode se aproveitar e comprometer o sistema por completo. Para evitar ue esse tipo de invasão aconteça, medidas simples de segurança podem ser tomadas por parte da organização como por exemplo:
- O uso de senhas fortes por parte de todos os usuários do sistema;
- Manter os serviços atualizados e bem configurados;
- Destativar os serviços que não estão sendo utilizados pelo sistema;
- Utilizar de algum tipo de sistema de verifcação multifatorial;
- Investir em sistemas de monitoramento inteligente;
- Entre outras formas de aumentar a segurança do sistema.
